# Release Notes: v0.1.0

`v0.1.0` is the first polished public release of ChurnOps as a portfolio-ready MLOps repository.

## What This Release Demonstrates

- a staged, test-backed churn training pipeline
- experiment tracking and best-model registration with MLflow
- a production-minded FastAPI inference service with metrics and health probes
- drift-aware monitoring that can trigger retraining through Airflow
- Docker and Kubernetes deployment paths that share the same runtime contract

## Recommended Demo Path

For the strongest repo walkthrough, use this sequence:

1. run `make train-fixture` to produce a local artifact bundle
2. run `make serve` and call `/health`, `/v1/model/metadata`, and `/v1/predictions`
3. inspect `/metrics` or start the local platform with `make platform-up`
4. review `docs/architecture.md` to explain how the layers fit together
5. render Kubernetes assets with `make k8s-render-staging` and `make k8s-render-production`

## Release Quality Gates

The repository is intended to pass the following local checks before release:

- `make lint`
- `make test`
- `make verify`

## Notes

- the current inference deployment remains single-replica because artifact and drift state still rely on local persistent storage
- MLflow may emit schema warnings about integer columns with missing values during tests; those warnings are expected with the current sample data
